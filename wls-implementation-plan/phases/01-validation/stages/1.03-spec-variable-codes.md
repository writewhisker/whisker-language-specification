# Stage 1.03: Specification - Variable Error Codes

## Objective
Add Variable (VAR) error codes to the validation specification.

## Files to Modify
- `phase-1-specification/spec/11-VALIDATION.md`

## Prompt

```
Update the validation specification at:
/Users/jims/code/github.com/whisker-language-specification-1.0/phase-1-specification/spec/11-VALIDATION.md

Add a new section for Variable (VAR) category after the Link section.

Include these error codes:

WLS-VAR-001: undefined_variable
- Severity: error
- Message: "Variable '{name}' is used but never defined"
- Example: Using $score without setting it

WLS-VAR-002: unused_variable
- Severity: warning
- Message: "Variable '{name}' is defined but never used"
- Example: Setting $temp but never reading it

WLS-VAR-003: invalid_variable_name
- Severity: error
- Message: "Invalid variable name '{name}'"
- Example: Using $123 or $my-var (invalid chars)
- Note valid pattern: [a-zA-Z_][a-zA-Z0-9_]*

WLS-VAR-004: type_mismatch_assignment
- Severity: warning
- Message: "Variable '{name}' assigned different types"
- Example: $x = 5 then $x = "hello"

WLS-VAR-005: temp_variable_scope
- Severity: error
- Message: "Temp variable '{name}' used outside defining passage"
- Example: $_local used in different passage

WLS-VAR-006: variable_shadowing
- Severity: warning
- Message: "Variable '{name}' shadows existing variable"
- Example: Local $_x when $x exists

WLS-VAR-007: reserved_variable_name
- Severity: error
- Message: "'{name}' is a reserved variable name"
- List: $whisker, $story, $passage, $choice, $visited

WLS-VAR-008: uninitialized_read
- Severity: warning
- Message: "Variable '{name}' may be read before initialization"
- Example: Conditional path where variable isn't always set

Format each consistently with previous sections.
Update table of contents.
```

## Expected Output
~120 additional lines added to the specification.

## Verification

```bash
grep -c "WLS-VAR-00" phase-1-specification/spec/11-VALIDATION.md
# Expected: 8

# Check section header
grep "## Variable" phase-1-specification/spec/11-VALIDATION.md
```
