# Stage 1.04: Specification - Expression Error Codes

## Objective
Add Expression (EXP) error codes to the validation specification.

## Files to Modify
- `phase-1-specification/spec/11-VALIDATION.md`

## Prompt

```
Update the validation specification at:
/Users/jims/code/github.com/whisker-language-specification-1.0/phase-1-specification/spec/11-VALIDATION.md

Add a new section for Expression (EXP) category after the Variable section.

Include these error codes:

WLS-EXP-001: empty_expression
- Severity: error
- Message: "Empty expression in conditional block"
- Example: {} or {do }

WLS-EXP-002: unmatched_parenthesis
- Severity: error
- Message: "Unmatched parenthesis in expression"
- Example: {($x + 1}

WLS-EXP-003: invalid_operator
- Severity: error
- Message: "Invalid operator '{op}' in expression"
- Example: {$x === $y} (should be ==)
- List valid operators: +, -, *, /, %, ==, !=, <, >, <=, >=, and, or, not

WLS-EXP-004: malformed_string
- Severity: error
- Message: "Unterminated string literal"
- Example: {$name == "hello}

WLS-EXP-005: invalid_number
- Severity: error
- Message: "Invalid number format '{value}'"
- Example: {$x = 1.2.3}

WLS-EXP-006: nested_expression_limit
- Severity: warning
- Message: "Expression nesting exceeds limit ({depth} > {max})"
- Example: Deeply nested conditions
- Default max: 10

WLS-EXP-007: unclosed_block
- Severity: error
- Message: "Unclosed conditional block (missing {/})"
- Example: {$x > 0} content without {/}

Format each consistently with previous sections.
Update table of contents.
```

~100 additional lines added to the specification.
