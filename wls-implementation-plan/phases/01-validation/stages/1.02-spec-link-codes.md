# Stage 1.02: Specification - Link Error Codes

## Objective
Add Link (LNK) error codes to the validation specification.

## Files to Modify
- `phase-1-specification/spec/11-VALIDATION.md`

## Prompt

```
Update the validation specification at:
/Users/jims/code/github.com/whisker-language-specification-1.0/phase-1-specification/spec/11-VALIDATION.md

Add a new section for Link (LNK) category after the Structure section.

Include these error codes:

WLS-LNK-001: dead_link
- Severity: error
- Message: "Link to undefined passage '{name}'"
- Example: Choice pointing to nonexistent passage

WLS-LNK-002: self_link
- Severity: warning
- Message: "Passage '{name}' links to itself"
- Example: Passage with choice back to same passage

WLS-LNK-003: special_target_case
- Severity: warning
- Message: "Special target '{name}' should be '{correct}'"
- Example: Using "end" instead of "END"
- List special targets: START, END, BACK, RETURN

WLS-LNK-004: empty_choice_target
- Severity: error
- Message: "Choice has no target passage"
- Example: Choice with empty ->

WLS-LNK-005: circular_link_chain
- Severity: info
- Message: "Circular link chain detected: {path}"
- Example: A -> B -> C -> A with no choices

Format each code consistently with the STR codes:
- Code and ID
- Severity level
- Message format
- Description
- Example WLS code
- Example error output

Update the table of contents to include LNK section.
```

~80 additional lines added to the specification.
