# Stage 1.01: Specification - Structure Error Codes

## Objective
Create the validation specification document with Structure (STR) error codes.

## Files to Create
- `phase-1-specification/spec/11-VALIDATION.md`

## Prompt

```
Create the WLS validation specification document at:
/Users/jims/code/github.com/whisker-language-specification-1.0/phase-1-specification/spec/11-VALIDATION.md

Include the following sections:

1. Overview of WLS validation system
2. Error code format: WLS-{CATEGORY}-{NUMBER}
3. Severity levels: error, warning, info
4. Structure (STR) category with these codes:

WLS-STR-001: missing_start_passage
- Severity: error
- Message: "Story must have a 'Start' passage"
- Example of invalid story without Start passage

WLS-STR-002: unreachable_passage
- Severity: warning
- Message: "Passage '{name}' is not reachable from Start"
- Example showing unreachable passage

WLS-STR-003: duplicate_passage
- Severity: error
- Message: "Duplicate passage name '{name}'"
- Example with two passages named the same

WLS-STR-004: empty_passage
- Severity: warning
- Message: "Passage '{name}' has no content"
- Example of empty passage

WLS-STR-005: orphan_passage
- Severity: info
- Message: "Passage '{name}' has no incoming links"
- Example of passage only reachable via Start

WLS-STR-006: bottleneck_passage
- Severity: info
- Message: "All paths pass through '{name}'"
- Example showing story bottleneck

Format each error code with:
- Code and ID
- Severity level
- Message format (with placeholders)
- Description
- Example WLS code that triggers it
- Example error output

Keep the document well-structured with a table of contents.
```

## Expected Output
~200 lines markdown file with STR error codes fully documented.

## Verification
```bash
# File exists and has content
wc -l phase-1-specification/spec/11-VALIDATION.md
# Expected: ~200 lines

# Contains all STR codes
grep -c "WLS-STR-00" phase-1-specification/spec/11-VALIDATION.md
# Expected: 6
```
