# Stage 5.06: Documentation Site

## Objective
Set up VitePress documentation site.

## Files to Create
- `docs/.vitepress/config.ts`
- `docs/index.md`
- `docs/getting-started/installation.md`
- `docs/getting-started/quick-start.md`

## Prompt

```
Set up VitePress documentation site:

1. Create docs/.vitepress/config.ts:

import { defineConfig } from 'vitepress'

export default defineConfig({
  title: 'Whisker Language',
  description: 'Interactive fiction language specification',

  themeConfig: {
    nav: [
      { text: 'Guide', link: '/getting-started/' },
      { text: 'Tutorials', link: '/tutorials/' },
      { text: 'API', link: '/api/' },
      { text: 'Examples', link: '/examples/' }
    ],

    sidebar: {
      '/getting-started/': [
        {
          text: 'Getting Started',
          items: [
            { text: 'Installation', link: '/getting-started/installation' },
            { text: 'Quick Start', link: '/getting-started/quick-start' },
            { text: 'Editor Setup', link: '/getting-started/editor-setup' }
          ]
        }
      ],
      '/tutorials/': [
        {
          text: 'Beginner',
          items: [
            { text: 'Your First Story', link: '/tutorials/beginner/01-first-story' },
            // ... more
          ]
        }
      ]
    },

    socialLinks: [
      { icon: 'github', link: 'https://github.com/writewhisker' }
    ],

    search: {
      provider: 'local'
    }
  }
})

2. Create docs/index.md (landing page):

---
layout: home
hero:
  name: Whisker Language
  text: Write interactive fiction
  tagline: A clean, powerful language for branching narratives
  actions:
    - theme: brand
      text: Get Started
      link: /getting-started/
    - theme: alt
      text: View on GitHub
      link: https://github.com/writewhisker
features:
  - title: Simple Syntax
    details: Easy to learn, familiar to writers
  - title: Powerful Features
    details: Variables, conditions, data structures
  - title: Cross-Platform
    details: Run in browsers, games, and apps
---

3. Create getting-started pages.

4. Build and preview:

cd docs
pnpm add -D vitepress
pnpm vitepress dev
pnpm vitepress build
```

## Expected Output
~200 lines of configuration + content.

## Verification
```bash
cd docs
pnpm vitepress build
# Should complete without errors
```

## Phase 5 Complete
Documentation infrastructure ready.
